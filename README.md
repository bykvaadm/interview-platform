# Опросник для собеседований (DevOps / SRE)

Статическое веб-приложение для проведения технических собеседований: банк вопросов, подсчёт баллов и определение грейда, совместная работа двух интервьюеров, чат, просмотр резюме с комментариями и выгрузка результатов в Excel/PDF. Перенос исходной Google-таблицы в «нормальный» веб-интерфейс; логика оценки грейда сохранена 1-в-1.

Весь код — в одном файле **`index.html`**. Никакого бэкенда: чистая статика на GitHub Pages. Живая версия: **https://bykvaadm.github.io/interview-platform/**

## Возможности

- **Подготовка** — банк вопросов с фильтрами по группе (CORE / MEDIUM / SPECIFIC), категории и сложности + поиск; набор вопросов в сессию. Готовые наборы (пресеты), включая «Базовый набор (как в таблице)».
- **Собеседование** — карточки вопросов: галочки «ключевых точек», оценка рекрутёра, корр. баллы; live-подсчёт суммы, вычисляемый грейд и визуальная шкала порогов. Комментарии по каждому вопросу и общий; сворачивание карточек.
- **📄 Резюме** — загрузка PDF-резюме и просмотр прямо в приложении (PDF.js). Выделите фрагмент → «💬 Комментарий»: комментарий с цитатой и номером страницы сохраняется и попадает в выгрузку.
- **👥 Совместная работа + чат** — два интервьюера работают над одной сессией в реальном времени (WebRTC P2P, без сервера): синхронизация опроса, резюме-комментариев и текстовый чат со звуком. Диагностика соединения (ICE-кандидаты, mDNS, «раскрыть реальный IP») и поле для своего TURN.
- **Банк и матрицы** — полностью редактируемые (вопросы, категории, матрицы баллов и ожиданий). Сохранение в localStorage; сброс к исходному; импорт/экспорт всей конфигурации одним JSON (вкладка «Подготовка» → «Данные»).
- **Экспорт** — Excel (.xlsx: «Итог», «Результаты», «Резюме») и PDF (оформленный отчёт с оценками и комментариями к резюме).
- **Светлая / тёмная** тема; адаптив (на узких экранах меню сворачивается в бургер).
- **Версионирование** — номер версии в шапке (клик → история изменений), источник правды — `CHANGELOG.md`.

## Как считается грейд

За каждый **оценённый** вопрос: `балл = матрица_баллов[степень раскрытия][сложность] + корр. баллы`.
Пороги грейдов считаются по набору оценённых вопросов и матрице ожиданий: для каждого грейда суммируется ожидаемый балл по всем вопросам. Сумма кандидата сравнивается с порогами → ближайший грейд и интервал. Неоценённые вопросы в расчёт не входят. Проверено: для набора 10 Low / 15 Medium / 4 High пороги совпадают с исходной таблицей — `12.5 / 44 / 59 / 67 / 75`.

## Совместная работа (P2P) и чат

Соединение прямое, между браузерами, без сервера (WebRTC, шифрование DTLS). В окне 👥:
1. Организатор: «Создать сессию» → копирует код-приглашение → шлёт коллеге.
2. Коллега: «Подключиться» → вставляет код → генерирует код-ответ → шлёт обратно.
3. Организатор вставляет код-ответ — готово.

Синхронизируются правки опроса, комментарии к резюме и чат. У каждого своё локальное «Ваше имя» для подписи в чате; поле «Интервьюер(ы)» — на двоих.

**Про сеть:** для прямого P2P между разными машинами нужен путь, который проходит через NAT/VPN. Chrome по умолчанию прячет локальный IP как `*.local` (mDNS) — между двумя ПК через VPN это часто не резолвится. Варианты: кнопка «🎤 Раскрыть реальный IP» (доступ к микрофону раскрывает реальный IP), флаг `chrome://flags/#enable-webrtc-hide-local-ips-with-mdns` → Disabled, либо свой TURN (поле «Свой TURN», напр. coturn на хосте, доступном обоим). Диагностика в том же окне показывает host/srflx/relay-кандидатов.

## Запуск локально

Просто откройте `index.html`. Для экспорта, P2P и просмотра PDF нужен интернет (библиотеки `xlsx`, `html2pdf`, `simple-peer`, `pdf.js` грузятся с CDN jsdelivr).

## Деплой на GitHub Pages

Приложение — статика в корне репозитория. GitHub Pages: **Settings → Pages → Deploy from a branch**, ветка `main`, папка `/ (root)`. Через ~1–2 минуты доступно по `https://<user>.github.io/<repo>/`.

## Разработка

`index.html` собирается из человекочитаемых исходников в `source/`:

```
source/
  questions.yaml  # категории (с группами CORE/MEDIUM/SPECIFIC) и банк вопросов
  presets.yaml    # готовые наборы (пресеты), ссылаются на вопросы по тексту
  build.py        # сборка index.html: YAML + матрицы + версия из CHANGELOG.md -> template.html
  template.html   # разметка/стили/логика (с плейсхолдерами данных/версии)
  test.js         # автотесты движка оценки и UI (node + jsdom)
  gen_yaml.py     # служебный: пересоздать YAML из predefined.json
```

Пересборка и тесты:

```bash
cd source
pip install pyyaml          # зависимость build.py (или: pip install pyyaml --break-system-packages)
python3 build.py            # пишет ../index.html  (переопределить путь: OUT_DIR=/path python3 build.py)
npm i jsdom && node test.js # прогон автотестов
```

**Версия и changelog:** источник правды — `CHANGELOG.md` в корне (формат [Keep a Changelog](https://keepachangelog.com/ru/)). `build.py` берёт верхнюю версию и зашивает в страницу (бейдж + история во вкладке «Инструкция»). Каждое изменение — новая запись сверху; версия на странице подтягивается сама.
