# Опросник для собеседований (DevOps / SRE)

Статическое веб-приложение — перенос вашей Google-таблицы для собеседований в нормальный веб-интерфейс. Логика подсчёта баллов и определения грейда сохранена 1-в-1 (проверено: пороги `12.5 / 44 / 59 / 67 / 75` совпадают с исходным примером).

Весь код — в одном файле **`index.html`**. Никакого бэкенда, сборки и зависимостей при запуске (две библиотеки для экспорта подгружаются с CDN). Можно просто открыть файл в браузере или выложить на GitHub Pages.

## Возможности

- **Подготовка** — банк вопросов с фильтрами по группе (CORE / MEDIUM / SPECIFIC), категории и сложности + поиск. Набираете вопросы в сессию.
- **Готовые наборы (пресеты)** — загрузка предустановленного набора вопросов одной кнопкой. Встроен пресет **«Базовый набор (как в таблице)»** — те 29 вопросов, что были выбраны в исходной Google-таблице. Любой свой выбор можно сохранить как новый пресет.
- **Собеседование** — карточки вопросов: галочки «ключевых точек», оценка рекрутёра, корр. баллы, балл за вопрос. Live-подсчёт суммы, вычисляемый грейд, визуальная шкала с порогами.
- **Банк вопросов** — полностью редактируемый: добавление / изменение / удаление вопросов и категорий.
- **Матрицы** — редактирование матрицы баллов и матрицы ожиданий по грейдам; экспорт/импорт всей конфигурации в JSON; сброс к исходному набору.
- **Экспорт** — кнопки **Excel** (.xlsx, два листа: «Итог» и «Результаты») и **PDF** (оформленный отчёт, корректная кириллица).
- **Светлая / тёмная** тема (переключатель в шапке).
- **Сохранение в браузере** (localStorage): банк, матрицы и текущая сессия не теряются при перезагрузке.

## Как считается грейд

За каждый **оценённый** вопрос: `балл = матрица_баллов[степень раскрытия][сложность] + корр. баллы`.
Пороги грейдов считаются по набору оценённых вопросов и матрице ожиданий: для каждого грейда суммируется ожидаемый балл по всем вопросам. Сумма кандидата сравнивается с порогами → ближайший грейд и интервал (нижний/верхний). Неоценённые вопросы в расчёт не входят (аналог «удаления неоценённых строк» в исходной таблице).

## Запуск локально

Просто откройте `index.html` двойным кликом. Для кнопок экспорта нужен интернет (CDN-библиотеки `xlsx` и `html2pdf`).

## Деплой на GitHub Pages

1. Создайте репозиторий и положите в него `index.html` (в корень).
   ```bash
   git init
   git add index.html
   git commit -m "interview questionnaire app"
   git branch -M main
   git remote add origin git@github.com:<user>/<repo>.git
   git push -u origin main
   ```
2. В репозитории: **Settings → Pages → Build and deployment → Source: Deploy from a branch**, ветка `main`, папка `/ (root)`. Сохранить.
3. Через ~1 минуту приложение будет доступно по адресу `https://<user>.github.io/<repo>/`.

> Так как это статика, состояние хранится в localStorage конкретного браузера. Чтобы перенести наполнение на другое устройство — «Матрицы → Экспорт конфигурации (JSON)», затем импорт на другом устройстве.

## Изменить стартовый набор вопросов

Стартовый банк зашит в `index.html`, но собирается из человекочитаемых YAML-файлов. Можно редактировать прямо в приложении (вкладки «Банк»/«Матрицы»), а можно — в исходниках и пересобрать:

```
source/
  questions.yaml  # ← категории (с группами CORE/MEDIUM/SPECIFIC) и банк вопросов
  presets.yaml    # ← готовые наборы (пресеты), ссылаются на вопросы по тексту
  build.py        # сборка index.html: YAML + матрицы оценок -> template.html
  template.html   # разметка/стили/логика (с плейсхолдером данных)
  test.js         # автотесты движка оценки и UI (node + jsdom)
  gen_yaml.py     # служебный: пересоздать YAML из predefined.json
```

Формат `questions.yaml`:

```yaml
categories:
  CORE:    [Linux, Helm, Kubernetes, ...]
  MEDIUM:  [Networking, S3, ...]
  SPECIFIC: [Openshift, Hashicorp Vault, ...]
questions:
  Linux:
    - level: Low          # Low | Medium | High
      q: Что такое systemd и зачем он нужен
      keys:               # ключевые точки — подсказки интервьюеру (0–3 шт.)
        - знает что systemd это подсистема инициализации
        - знает про service, timer, socket
```

`presets.yaml` ссылается на вопросы по тексту (build.py сопоставляет по паре категория + текст):

```yaml
presets:
  - id: builtin-base
    name: Базовый набор (как в таблице)
    builtin: true
    questions:
      Linux: [Что такое systemd и зачем он нужен, Какие права на файл бывают]
      Helm:  [структура helm пакета]
```

Пересборка и тесты:

```bash
cd source
pip install pyyaml          # зависимость build.py (или: pip install pyyaml --break-system-packages)
python3 build.py            # пишет ../index.html  (переопределить путь: OUT_DIR=/path python3 build.py)
npm i jsdom && node test.js # прогон автотестов
```
